# MistTerm — 功能设计规范

---

## 1. 会话管理（连接）

### 1.1 背景

MistTerm 的核心场景是管理多个 SSH 服务器的连接。用户在日常运维中通常需要同时维护开发、测试、预发布、生产等多个环境，频繁切换不同服务器。

手动输入 `ssh user@host -p port` 在终端数量多时效率极低，且密码/密钥管理困难。

### 1.2 设计目标

- **单点管理**：所有 SSH 连接在一个面板中统一管理，无需离开应用
- **零配置连接**：保存的连接一键连接，无需重复输入凭据
- **状态可视**：每台服务器的连接状态一目了然（在线/离线）
- **快速切换**：支持搜索过滤，大数量级（50+）下仍能秒级定位

### 1.3 功能细节

#### 1.3.1 会话数据结构

```json
{
  "id": "uuid",                 // 唯一标识，创建时生成
  "name": "生产服务器",          // 用户自定义显示名
  "host": "192.168.1.100",      // IP 或域名
  "port": 22,                   // SSH 端口
  "username": "root",           // 登录用户名
  "password": "<encrypted>",    // 本地 AES 加密，不同步至团队服务器
  "group": "默认分组",          // 分组标签
  "proxy_jump": "",             // OpenSSH ProxyJump（逗号多跳；可匹配已保存会话名）
  "proxy_command": "",          // OpenSSH ProxyCommand（%h/%p/%r/%u）
  "local_forwards_text": "",    // 本地端口转发 (-L)，每行 local:remote_host:remote_port
  "remote_forwards_text": "",   // 远程端口转发 (-R)，每行 remote:target_host:target_port
  "dynamic_forwards_text": "",  // 动态 SOCKS (-D)，每行 port 或 bind:port
  "last_connected_at": 0,       // 上次连接时间戳 (秒)
  "created_at": 0,              // 创建时间
  "updated_at": 0               // 更新时间
}
```

**数据约束：**
- `id`: 使用 uuid v4，创建时初始化后不可修改
- `port`: 默认 22，合法范围 1–65535
- `password`: 使用 AES-GCM 加密（AES-256），密钥来源于系统 keyring
- `group`: 默认值为 "默认分组"，不能为空
- `last_connected_at`: 0 表示从未连接过，显示为 "未连接"

#### 1.3.2 新建会话

**触发方式：**
1. 左侧面板底部 "新建连接" 按钮
2. 快捷键 `⌘N` 或 `Ctrl+N`

**表单字段：**

| 字段 | 类型 | 是否必填 | 默认值 | 校验 |
|---|---|---|---|---|
| 服务器名称 | text | 是 | 空 | 1–50 字符，不能纯空格 |
| 主机地址 | text | 是 | 空 | IP / 域名格式 |
| 端口 | number | 是 | 22 | 1–65535 |
| 用户名 | text | 是 | 空 | 1–50 字符 |
| 密码 | password | 是 | 空 | AES-GCM 加密存储 |
| 分组 | select | 是 | "默认分组" | 可选已有分组或输入新分组 |

**校验规则：**
- 主机地址：必须匹配 IP (`\d+\.\d+\.\d+\.\d+`) 或域名格式，不校验可达性
- 所有必填字段为空时，提交按钮置灰
- 密码字段不显示明文，使用 `password` 类型

**保存策略：**
- 校验通过后写入 `~/.config/mistterm/sessions.json` 或同级 `configs.json`
- 写入失败（权限/磁盘满）：弹窗提示错误
- 保存后自动刷新左侧列表

#### 1.3.3 编辑会话

**触发方式：**
1. 右键点击连接条目 → "编辑"
2. 选中条目后快捷键 `⌘E`

**约束：**
- 编辑时预填现有值
- 密码字段预填 "****"（不泄露密码），用户可重新输入
- 修改后保存到文件
- 如果该会话当前已连接，允许编辑，但连接不会断开，新连接不会收到变更

#### 1.3.4 删除会话

**触发方式：**
1. 右键点击连接条目 → "删除"
2. 选中后按 `Delete` 键

**确认：**
- 弹出确认对话框："确认删除 [名称] 的会话配置？此操作不可恢复。"
- 确认后从文件移除，若当前已连接 → 先断开再删除

#### 1.3.5 连接/断开

**连接流程：**
1. 从文件读取会话的 host/port/username/password
2. 密码解密（AES-GCM + device_key）
3. 若配置了 `proxy_jump` / `proxy_command`，按 OpenSSH 语义建立跳板或子进程代理
4. 校验 SSH 主机密钥（`~/.config/mistterm/known_hosts`）：未知主机提示信任，指纹变更则拒绝
5. 建立 SSH 会话（ssh2）
6. 连接成功后按需启动端口转发：
   - `local_forwards_text` → `-L`
   - `remote_forwards_text` → `-R`
   - `dynamic_forwards_text` → `-D`（本地 SOCKS5，经 SSH `direct-tcpip` 转发）
7. 连接成功后：
   - 更新 `last_connected_at` 为当前时间戳
   - 左侧圆点变为绿色
   - 终端区域创建新 Tab，标题显示 `username@host`
   - 状态栏显示连接信息
8. 连接失败：
   - 弹窗提示具体失败原因（超时/认证失败/拒绝连接）
   - 左侧圆点保持灰色/红色
   - 终端区域显示错误信息（仅当前 Tab，不影响其他 Tab）

**断开流程：**
1. 关闭对应 SSH session
2. 释放 channel
3. 更新左侧圆点为离线
4. Tab 仍然保留（可手动关闭）
5. 断开后终端内容冻结（不可继续输入）

#### 1.3.6 搜索与过滤

- 搜索框输入后实时过滤连接列表（匹配名称/主机/分组）
- 分类标签："全部 / 在线 / 离线"
- 搜索和分类标签可以组合使用

#### 1.3.7 分组管理

- 新建/编辑会话时可选择或输入分组
- 列表按分组折叠显示（同一分组下的条目相邻）
- 分组不是文件夹层级结构，仅做视觉分组

### 1.4 边界条件

| 场景 | 行为 |
|---|---|
| 同时连接 10+ 台服务器 | 所有连接各自运行在独立 tokio task 中，支持并行连接 |
| 服务器超时无响应（>30s）| 连接失败，提示 "连接超时" |
| 认证凭据错误 | 提示 "认证失败，请检查用户名/密码" |
| 服务器拒绝连接（端口未开放） | 提示 "连接被拒绝" |
| 网络断开后重连 | 默认不自动重连；可在偏好设置或会话编辑中启用「断开后自动重连」（意外断开时指数退避重试，手动断开不重连） |
| 会话文件中密码字段损坏 | 提示 "密码数据损坏，请重新编辑会话" |
| 会话文件正在被其他进程写入 | 读取失败时重试 3 次，间隔 100ms，仍失败则提示错误 |
| 主机域名 DNS 解析失败 | 提示 "无法解析主机地址" |

## 2. 终端模拟器

### 2.1 背景

用户通过 SSH 连接后需要在一个类 xterm 的终端环境中执行命令、查看输出。纯 Rust 实现依赖 `alacritty_terminal` crate 提供 VTE (VT100/ANSI) 解析能力和网格 (grid) 屏幕状态管理。

### 2.2 设计目标

- **语义精确**：正确解析 VT100/ANSI 转义序列，支持 256 色和 truecolor
- **滚动保留**：保留历史输出，支持通过滚动条回看
- **语法着色**：命令输出中区分 prompt、路径、命令、输出内容，便于阅读
- **响应式**：SSH 输出源源不断时保持渲染流畅（每秒 30+ fps）

### 2.3 功能细节

#### 2.3.1 渲染流程

```
SSH 输出字节 → vte::Parser (VT100 解析) → alacritty_terminal::Term (网格更新)
  → get_layout_job() 生成着色布局 → egui::Painter::galley()
```

- `vte::Parser` 消费 SSH 字节流，更新终端状态机
- `alacritty_terminal::Term` 维护光标位置、网格单元、滚动区域
- `TermView` (自定义) 将网格数据转换为 egui 可渲染的 `LayoutJob`（每一行一个 `Galley`）
- 当终端输出更新时，仅重绘有变化的行

#### 2.3.2 语法着色规则

| 行首标记 | 颜色 | 说明 |
|---|---|---|
| `➜` (U+279C) | `#4CAF50` (绿色) | Prompt 符号 |
| `~` 或路径 | `#667eea` (紫色) | 当前路径 |
| 命令正文 | `rgba(255,255,255,0.9)` | 用户输入的命令 |
| 输出行 (非prompt行) | `rgba(255,255,255,0.4)` | 命令行输出结果 |
| ANSI 转义指定色 | 由 alacritty_terminal 解析输出 | 如 ls --color 的输出颜色 |

**着色优先级**：ANSI 转义序列指定色 > 语法标注色

#### 2.3.3 输入处理

- 用户在当前行末尾输入（contenteditable 模式）
- 按 `Enter` → 发送输入字节到 SSH channel
- 输入行不清空，追加到输出历史中
- 支持复制/粘贴（选中内容后 `⌘C`/`Ctrl+C`）

#### 2.3.4 滚动

- 使用 egui 原生 `ScrollArea`
- 新输出自动滚动到底部（跟随）
- 用户手动向上滚动后，锁定滚动位置（不跟随）
- 用户滚动到底部时恢复跟随
- 滚动条宽度 4px，颜色 `rgba(255,255,255,0.06)`

#### 2.3.5 多 Tab

- 每个 Tab 持有独立的 `TerminalView`（含独立的 SSH 连接和终端模拟器）
- Tab 名称：`username@host.tld` (从连接名称推导)
- Tab 前圆点：绿色 = 连接中，灰色 = 已断开
- 关闭 Tab：点击 `×`，确认（如果连接活跃时）→ 断开连接并移除 Tab
- 新建 Tab：点击 `+`，弹出会话选择列表

**Tab 快捷键：**
| 快捷键 | 行为 |
|---|---|
| `⌘T` / `Ctrl+T` | 新建 Tab |
| `⌘W` / `Ctrl+W` | 关闭当前 Tab |
| `⌘{数字}` / `Ctrl+{数字}` | 切换到第 N 个 Tab |
| `⌘Tab` / `Ctrl+Tab` | 切换到下一个 Tab |

#### 2.3.6 Tab 内分屏（已实现）

同一 Tab 内可并排或上下显示多个终端窗格（共享 Tab 标签，各自独立 SSH 会话）。

| 能力 | 说明 |
|------|------|
| 布局 | 树形嵌套左右/上下分屏，最多 **8** 窗格 |
| 换位 | 拖拽窗格标题栏到另一窗格标题上松开，交换两窗格会话 |
| 触发 | 标签栏右键 →「左右分屏」「上下分屏」「合并分屏」「关闭窗格」；窗格标题栏 `×` 关闭 |
| 新窗格 | 分屏时复制当前窗格所连会话，**新建一条 SSH 连接**（非共享 channel） |
| 焦点 | 点击窗格聚焦；`Alt+←` / `Alt+→` 在分屏窗格间循环切换焦点 |
| 窄屏 | 终端列宽 **&lt; 480px** 时自动合并为单窗格（保留当前焦点窗格） |
| 市场目录 | 支持 `cursor` 分页「加载更多」 |

**边界：**
- 已达 8 窗格时不再允许继续分屏
- 仅剩 1 窗格时「关闭窗格」等同于无操作；「合并」恢复单窗格布局

#### 2.3.7 批量多机执行（已实现）

| 项 | 说明 |
|----|------|
| 入口 | **工具 → 批量多机执行…**（macOS 系统菜单同名项） |
| 目标 | 已保存的**个人会话**；勾选「包含团队服务器」时含当前团队同步的服务器列表 |
| 执行 | 每台主机**单独建连**，`exec` 一条命令（不占用终端标签、不开 shell） |
| 并行 | 可调最大并行数 1–16，按批在后台线程执行 |
| 结果 | 每台显示成功/失败、退出码、耗时、stdout；支持**复制结果**到剪贴板 |
| 审计 | 受团队**命令审计**策略约束：`block` 拒绝；`confirm` 需交互终端确认，批量不可用 |
| 连接 | 支持 `ProxyJump` 跳板链、Vault 凭据、端口转发字段（与单会话连接一致） |

### 2.4 边界条件

| 场景 | 行为 |
|---|---|
| SSH 会话断开后继续输入 | 输入不发送，本地保存到缓冲区，重连后提示是否重发 |
| 终端输出超过 10000 行 | 自动截断历史输出，保留最后 10000 行 |
| 窗口 resize（cols/rows 变化）| 调用 `Terminal::resize(cols, rows)` 触发 alacritty 重新布局 |
| ANSI 转义序列解析异常 | vte parser 静默跳过无法识别的序列，不崩溃 |
| 用户粘贴超长文本（>10KB）| 分批发送，每批 4096 字节，间隔 5ms |
| 同时打开 20+ Tab | 每个 Tab 独立渲染，非活跃 Tab 降低刷新率（不重绘网格）|

## 3. 命令片段管理

### 3.1 背景

运维和开发人员在日常工作中频繁执行大量相似命令，如 `kubectl logs -f`、`systemctl restart nginx`、`df -h` 等。每次手动输入不仅低效而且容易出错。这些命令应当像"代码片段"一样被收藏和复用。

### 3.2 设计目标

- **即点即用**：点击命令片段后自动填入终端输入行，无需手动复制粘贴
- **分类导航**：按类别组织（Docker/K8s/系统/网络），快速定位
- **团队协作**：支持团队共享片段，区分个人/团队/市场来源
- **使用统计**：记录每个片段的使用次数/成功率/耗时，帮助识别高效命令

### 3.3 功能细节

#### 3.3.1 数据结构

```json
{
  "id": "uuid",
  "title": "查看 Pod 日志",            // 片段标题
  "command": "kubectl logs -f --tail=100",  // 实际命令
  "tag": "team",                      // 标签: personal / team / market
  "categories": ["K8s", "日志"],      // 分类标签
  "usage_count": 320,                 // 使用次数
  "success_rate": 95.0,              // 成功率百分比
  "avg_time_ms": 1200,               // 平均执行耗时 (ms)
  "created_at": 0,
  "updated_at": 0,
  "created_by": "erp001"             // 创建人 ERP
}
```

**数据约束：**
- `tag`：仅限 `personal` / `team` / `market` 三个枚举值
- `success_rate`：0.0–100.0，保留一位小数
- `avg_time_ms`：0 表示未统计（新片段默认 0）
- `useage_count`：新增默认为 0

#### 3.3.2 卡片列表

- 每个卡片展示：标题 + tag标签 + 命令原文（单行截断）+ 统计数
- 分类标签栏：`常用 | Docker | K8s | 全部`
- 搜索框：实时模糊过滤（匹配标题和命令原文）
- 搜索和分类标签可组合使用

#### 3.3.3 点击插入

- 点击片段卡片 → 将 `command` 字符串插入当前终端输入行的末尾
- 如果当前无活跃 Tab 或多个 Tab 均未连接 → 提示 "请先选择一个连接"
- 插入后不自动执行（用户按 Enter 后执行）

#### 3.3.4 统计更新

- 每次命令执行后尝试判断是否成功：
  - 成功判断：退出码为 0 → 记为成功
  - 失败判断：退出码非 0 或连接断开 → 记为失败
  - 不明确（连接中途断开）：不记入统计
- 更新公式：
  - `useage_count += 1`
  - `success_rate = (success_count / usage_count) * 100`
  - `avg_time_ms = (avg_time_ms * (usage_count - 1) + current_time_ms) / usage_count`

#### 3.3.5 列表范围（个人 / 团队 / 市场）

右侧面板顶部 scope 切换：

| 范围 | 数据来源 | 说明 |
|------|----------|------|
| **个人** | 本地 `fragments.json` | 可编辑；展示个人 Top5 使用片段 |
| **团队** | 登录后 `fragments:sync` 缓存 | 按角色控制新建/编辑；409 冲突可解决 |
| **市场** | `GET /v1/market/fragments/catalog` + 本地缓存 | 只读浏览；右键「添加到个人库」；服务端 404 时用缓存 + 本地带 `market` 标签的片段 |

市场目录在后台线程拉取，不阻塞 UI；支持手动刷新。

#### 3.3.6 分析大盘（已实现 MVP）

入口：片段侧栏 **「分析大盘」** 按钮。

| 模块 | 内容 |
|------|------|
| KPI | 个人/团队执行次数、成功率、平均耗时 |
| 排行 | 个人 Top5、团队 Top5 |
| 诊断 | 最慢 Top5、高错误率 Top5（至少 3 次执行） |
| 服务端 | `GET /v1/teams/{id}/fragments/analytics`；404 时仅本地 + 团队缓存聚合 |

| 时间范围 | 全部 / 近 7·30·90 天；非「全部」时次数来自本机执行日志（增量） |
| 成员对比 | 区间内本机执行的团队片段按成员汇总（待服务端全量成员统计 API） |
| 导出 | 「导出 JSON」复制到剪贴板 |

完整愿景见 `fragments-analytics.md`；成员对比、区间内增量统计、效率报告 PDF 为后续版本。

#### 3.3.7 命令审计（团队，已实现）

登录团队且服务端启用命令审计时，**发送命令前**本地匹配策略（sync 缓存 + 内置规则）：

| 动作 | 行为 |
|------|------|
| block | 不发送，状态栏提示 |
| confirm | 弹窗确认后发送 |
| alert | 发送并 `POST .../command-audit/alerts` |

详见 `docs/tech/COMMAND-AUDIT.md`。

### 3.4 边界条件

| 场景 | 行为 |
|---|---|
| 搜索无结果 | 显示 "没有匹配的命令片段" |
| 命令含特殊字符（引号/换行）| 保持原样插入，不转义 |
| 命令长度超过 2000 字符 | 仍然正常显示，但全文在卡片中截断显示前 80 字符 + "…" |
| 统计文件损坏/丢失 | 保留命令数据，统计项重置为 0 |
| 点击后终端未连接 | 提示错误，不影响当前片段使用次数 |

## 4. 文件传输

### 4.1 背景

用户连接到远程服务器后，经常需要上传/下载文件。MistTerm 支持多种文件传输方式以适应不同场景：SCP（小文件/简单场景）、ZMODEM（交互式/兼容性好）、cat pipe（无额外依赖）。

### 4.2 设计目标

- **灵活通道**：根据文件大小和网络条件自动选择最优传输方式
- **中断续传**：大文件传输意外中断支持断点续传（ZMODEM 原生支持）
- **进度可见**：传输过程中实时显示进度、速率和剩余时间

### 4.3 功能细节

#### 4.3.1 传输方式选择

| 方式 | 适用场景 | 优点 | 限制 |
|---|---|---|---|
| SCP | 单文件传输，<500MB | 简单快捷，SSH 自带协议 | 不支持通配符，大文件无进度 |
| ZMODEM | 交互式传输，支持续传 | 中断续传，双向通道，兼容 rz/sz | 服务端需安装 lrzsz |
| cat pipe | 小文件应急传输 | 纯 SSH 通道，零依赖 | 仅上传，无进度显示 |

**自动选择规则：**
- 文件 <10MB → SCP（最快）
- 文件 ≥10MB 且服务器支持 ZMODEM → ZMODEM（有进度）
- 服务器无 lrzsz → SCP

#### 4.3.2 上传流程

1. 用户点击状态栏 `📤` 或拖拽文件到终端区域
2. 系统弹出文件选择对话框 (`rfd::FileDialog`)
3. 用户选择文件
4. 根据文件大小自动选择传输方式
5. 状态栏显示进度条/百分比/速率
6. 传输完成显示 "完成" 状态
7. 传输失败显示错误信息，ZMODEM 支持续传

#### 4.3.3 下载流程

1. 在终端中输入触发服务端发送的命令（如 `sz filename`）
2. MistTerm 捕获 ZMODEM 协商信号
3. 弹出本地保存对话框
4. 接收文件，显示进度
5. 传输完成后通知用户

### 4.4 边界条件

| 场景 | 行为 |
|---|---|
| 大文件传输中 SSH 断开 | SCP 传输失败无续传；ZMODEM 支持恢复 |
| 磁盘空间不足（本地/远程）| 提示 "磁盘空间不足" |
| 文件名含中文/特殊字符 | UTF-8 编码传输，不转码 |
| 上传目录不存在（远程）| SCP 自动创建目录失败时报错 |
| 同时发起多个传输 | 队列化，一次只允许一个传输 |

## 5. 凭据安全

### 5.1 背景

SSH 密码直接存储在配置文件中存在安全风险。MistTerm 管理着用户的多台服务器凭据，需要提供足够的安全保障。

### 5.2 设计目标

- **最小化明文暴露**：密码仅在连接时解密，其余时间以密文形式存储
- **设备级加密**：会话/片段/凭证等 JSON 使用 AES-256-GCM，密钥来自本机 `device_key`（指纹派生，见 `docs/tech/SECURITY.md`）
- **不上传云端**：凭据文件被 `.gitignore` 排除，不随文档同步

### 5.3 功能细节

**加密策略：**
1. 用户保存/更新密码时写入 `sessions.json` 信封（`mistterm-aes-v1` 或字段级 `encrypted_password` + `password_nonce`）
2. 密钥来自本机 `device_key`，换机需相同设备密钥才能解密
3. 连接时解密 → 传入 ssh2 库

**实现路径：**
```
保存: password → device_key → AES-GCM → sessions.json（加密信封或字段）
读取: sessions.json → device_key 解密 → password → ssh2::Session::userauth_password()
```

**GIT 保护：**
- `.gitignore` 已包含 `sessions.json` 和 `configs.json`
- CI 中不会误提交凭据

### 5.4 边界条件

| 场景 | 行为 |
|---|---|
| keyring 服务不可用（无 GUI 环境）| 回退至本地文件加密密钥 |
| AES 密钥文件被删除 | 无法解密任何密码，需用户重新输入所有密码 |
| 凭据文件明文泄漏（误操作） | 所有密码应立即修改，并在新版本中轮换加密密钥 |

## 7. 窗口与响应式

### 7.1 设计目标

- **最小工作宽度：800px** — 低于此宽度时自动隐藏右侧面板
- **面板折叠可恢复** — 用户折叠的面板在窗口缩放后不会自动展开
- **多显示器适应** — 窗口位置和大小在退出时保存并在下次启动恢复

### 7.2 响应式规则

| 窗口宽度 | 左侧面板 | 终端区域 | 右侧面板 |
|---|---|---|---|
| ≥1200px | 200px 展开 | flex:1 | 260px 展开 |
| 800–1199px | 200px 展开 | flex:1 | 自动折叠到状态栏 |
| <800px | 自动折叠到状态栏 | flex:1 | 自动折叠到状态栏 |

> **注意**：如果用户手动折叠了某面板，响应式规则不会自动展开它（尊重用户意图）。
