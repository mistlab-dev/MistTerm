## 一、为什么需要 MistTerm？

### 1.1 痛点分析

```
传统终端工具的问题：
┌─────────────────────────────────────────────────────────┐
│ ❌ 命令重复输入：常用命令每次都重新敲                    │
│ ❌ 工具割裂：终端、SFTP、监控要用不同工具                │
│ ❌ 团队经验无法沉淀：老员工离职，经验带走                 │
│ ❌ 新人上手慢：没人教，文档找不到                        │
│ ❌ 配置不同步：换电脑重新配一遍                          │
│ ❌ 凭证明文存储：不安全，还容易忘                        │
└─────────────────────────────────────────────────────────┘
```

### 1.2 MistTerm 的解决方案

```
MistTerm 的核心理念：一次配置，全员复用

┌─────────────────────────────────────────────────────────┐
│  张三（老员工）                                          │
│    ↓ 创建命令片段 + 共享到团队                            │
│  团队片段库                                              │
│    ↓ 李四/王五/新人一键采用                               │
│  全员效率提升 10 倍                                        │
└─────────────────────────────────────────────────────────┘
```

### 1.3 效率提升量化

| 场景 | 传统方式 | MistTerm | 效率提升 |
|-----|---------|---------|---------|
| 常用命令执行 | 每次输入 20 字符 | 快捷键调用 1 秒 | **10 倍** |
| 新人上手 | 1-2 周学习 | 1 天熟悉片段库 | **10 倍** |
| 文件传输 | 切换工具 + 手动操作 | 拖拽/命令一键 | **5 倍** |
| 服务器监控 | SSH 登录 + 查命令 | 实时面板 | **3 倍** |
| 配置同步 | 手动重新配置 | 云端自动同步 | **1 次配置** |

## 二、核心效率功能

### 2.1 命令片段（效率核心）

**一句话**：把常用命令存起来，一键执行。

#### 个人片段（基础）

```
场景：每天都要执行的命令

❌ 传统方式：
$ docker ps -a | grep nginx | awk '{print $1}' | xargs docker restart

✅ MistTerm：
1. 创建片段「重启 Nginx 容器」
2. 绑定快捷键 Ctrl+Shift+1
3. 以后按快捷键，1 秒完成
```

#### 团队片段（核心亮点）

```
场景：团队常用命令共享

张三创建 → 共享到运维团队 → 李四/王五/新人全部可用

优势：
- 新人快速上手，不用问老员工
- 统一操作规范，减少人为错误
- 经验沉淀，员工离职不带走
```

#### 版本控制

```
v1.0 → v1.1 → v1.2
  ↓       ↓       ↓
  可以回滚到任意版本，有变更记录
```

#### 评论讨论

```
片段：部署应用
────────────────────────────
张三：这个脚本在生产环境用过了吗？
李四：用过了，昨天刚部署过，没问题
王五：建议加个回滚脚本，以防万一
张三：好主意，v1.2 加上
```

### 2.2 模板市场（生态建设）

**一句话**：优质命令可以发布到市场，被更多团队采用。

```
运维团队 → 发布「K8s 常用命令」模板 → 100 个团队采用

价值：
- 团队经验被认可
- 全行业效率提升
- 形成良性生态
```

### 2.3 文件传输（效率提升 5 倍）

**一句话**：不用切换工具，终端内直接传文件。

#### SFTP 双面板

```
┌──────────────┬──────────────┐
│   本地文件    │   远程文件    │
│              │              │
│  [拖拽] ────>│              │
│              │  [拖拽] <────│
│              │              │
└──────────────┴──────────────┘
```

#### lrzsz 命令支持

```
终端输入：
$ rz                    # 上传文件
$ sz filename.txt       # 下载文件

MistTerm 自动弹出文件选择器，无需配置
```

### 2.4 凭证管理（安全 + 效率）

**一句话**：密码记在云端，安全还不用背。

```
❌ 传统方式：
- 密码记在脑子里（容易忘）
- 密码记在 Excel 里（不安全）
- 每次连接都要输入（麻烦）

✅ MistTerm：
- 加密存储，端到端加密
- 自动填充，一键连接
- 团队共享，权限可控
- 操作审计，可追溯
```

## 三、被砍掉/简化的功能

### 3.1 为什么砍掉某些功能？

**原则**：不直接提升效率的功能，要么砍掉，要么简化。

### 3.2 主题定制（简化）

**原计划**：深度自定义颜色、字体、布局、动画...

**简化后**：
- 提供 10 个内置主题（够用）
- 支持颜色微调（基础）
- **砍掉**：复杂布局自定义、动画效果（用不到）

**理由**：90% 的用户只需要换个颜色，不需要当设计师。

### 3.3 团队协作（合并）

**原计划**：独立的团队协作模块（邀请、权限、审计...）

**合并后**：
- 合并到「命令片段」模块
- 权限控制保留（必须）
- 审计日志保留（企业需要）
- **简化**：复杂的团队管理功能

**理由**：团队协作的核心是共享命令片段，其他都是辅助。

## 四、界面设计原则

### 4.1 少即是多

```
❌ 复杂界面：
┌──────────────────────────────────────────┐
│ [菜单 1] [菜单 2] [菜单 3] [菜单 4] [菜单 5]│
│ 子菜单 A  子菜单 B  子菜单 C  子菜单 D  子菜单 E│
│ ... 10 层菜单 ...                          │
└──────────────────────────────────────────┘

✅ 简洁界面：
┌──────────────────────────────────────────┐
│ [片段] [传输] [监控] [设置]              │
│                                          │
│         核心功能一目了然                  │
└──────────────────────────────────────────┘
```

### 4.2 快捷键优先

```
核心操作都必须有快捷键：
- 打开片段：Ctrl+J
- 新建片段：Ctrl+N
- 上传文件：Ctrl+U
- 下载文件：Ctrl+D
- 新建标签：Ctrl+T
- 关闭标签：Ctrl+W
```

### 4.3 智能推荐

```
场景：输入 docker 后

智能推荐：
1. docker ps（你常用）
2. docker logs（团队常用）
3. docker restart（昨天用过）
```
