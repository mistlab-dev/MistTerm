# MistTerm UI 改造设计规范

**交付对象**: Rust/egui 开发  
**当前分支代码**: `app.rs` + `theme.rs` + `chrome.rs` + `layout_util.rs`  
**设计原则**: 改造所有 `theme.rs` 的样式令牌 + 删除 app.rs 中菜单栏代码（不重写布局架构）

## 一、颜色体系（修改 theme.rs）

**Token v2（单一真源）**：业务用 `surface_*` / `panel_stroke()` / `divider_stroke()` / `text_primary|secondary|tertiary`，禁止 `fg_high_alpha(8~20)` 画面板边框。晨曦 / 海洋 / 森林边框见各 `Theme::*()` 实色或加强 alpha 表。

### 3.2 状态栏（app.rs show_bottom_chrome）

**当前**: 双层（44px 快捷栏 + 28px 紫色状态栏）  
**目标**: 32px 暗色单行

```rust
fn show_bottom_chrome(&mut self, ctx: &egui::Context) {
    let theme = self.theme_manager.current_theme().clone();
    let status_h = 32.0;  // theme.status_bar_height() → 从 28 改为 32
    
    egui::TopBottomPanel::bottom("bottom_chrome")
        .exact_height(status_h)
        .frame(egui::Frame::none().inner_margin(egui::Margin::ZERO))
        .show(ctx, |ui| {
            let rect = ui.max_rect();
            // 底色: rgba(255,255,255,0.04) 而不是 #667eea 紫色
            ui.painter().rect_filled(rect, 0.0, 
                Color32::from_rgba_unmultiplied(10,10,10,10));  // 4% 白
            // 顶部分割线
            ui.painter().hline(rect.x_range(), rect.top(),
                Stroke::new(1.0, theme.border_divider_color()));
            
            ui.add_space(14.0);  // theme.spacing_status_bar_x()
            ui.horizontal(|ui| {
                // 左侧: 复原按钮 + 连接信息
                ui.spacing_mut().item_spacing = egui::vec2(8.0, 0.0);  // theme.spacing_status_left_gap()
                
                // 复原按钮 (仅折叠时显示)
                if self.sidebar_collapsed {
                    // chrome.rs 已有 status_restore_chip
                }
                
                // 连接信息: 11px, 12% 透明度
                ui.label(RichText::new(conn_line)
                    .size(11.0)  // theme.font_size_status_bar()
                    .color(Color32::from_rgba_unmultiplied(255,255,255,31)));  // 12%
                    
                // 右侧: 工具按钮 (4个) + 分隔线 + 统计

                    ui.spacing_mut().item_spacing = egui::vec2(3.0, 0.0);  // spacing_tool_btn_gap
                    
                    // 统计数字: 10px
                    ui.label(RichText::new("1,234次")
                        .size(10.0)
                        .color(Color32::from_rgba_unmultiplied(255,255,255,64)));  // 25%
                    
                    // 分隔线 |
                    ui.add_space(4.0);  // spacing_status_right_gap
                    ui.label("|".size(10).color(rgba(255,255,255,10)));
                    
                    // 4 个工具按钮: 默认8% hover 25%
                    // 用 status_tool_glyph(ui, &theme, "📊") — chrome.rs 已有
                    // 顺序: 📊监控 → 🔍搜索 → 📤上传 → 📋片段
                });
            });
        });
}
```

## B.3 值得做的高价值功能（按 ROI 排序）

### 🥇 P0 — 投入小、收益大（1-2 天）

**1. SSH Config 导入（~1天）**
- 解析 `~/.ssh/config` → 自动导入为会话
- 几乎每个竞品都有，是用户迁移的第一需求

**2. 命令执行历史搜索 / Ctrl+R（~1天）**
- `fragment_manager.record_execution` 已记录了次数/成功率
- 新建一个 `RecentCommands` 面板，按时间倒序展示最近命令
- Ctrl+R 打开搜索框

**3. 终端输出日志自动保存（~0.5天）**
- `~/.config/mistterm/logs/{session_id}/YYYY-MM-DD.log`
- 可开关，配置面板加勾选项

**4. KeepAlive/ServerAliveInterval 配置（~0.5天）**
- SSH 连接断开恢复极常用
- 会话编辑面板加一个"心跳保活"开关（默认开，间隔 30s）

### 🥈 P1 — 中等投入、显著提升体验（3-7 天）

**5. Tab 分屏（~3天）**
- 每个 Tab 内可左右/上下分屏
- 同时看日志+监控 刚需

**6. 代理链 / 跳板机（~3天）**
- `ProxyJump` + `ProxyCommand`
- 会话编辑新增"代理类型：直连 / 跳板机 / SSH隧道"

**7. 批量多机执行（~5天）**
- 多选连接→底部弹"批量执行"框
- 命令并发发送，输出逐台展示

**8. SSH 端口转发（~4天）**
- 本地/远程/动态三种转发
- 独立隧道管理面板

**9. 会话排序 & 颜色标签（~2天）**
- 按名称/连接时间/主机排序排序
- 给连接加颜色标记（红/黄/绿/蓝/紫）

### 🥉 P2 — 差异化亮点（5-10 天）

**10. AI 命令助手（~5天）**
- 自然语言→Shell 命令（调 Ollama / OpenAI 兼容 API）
- 与 Warp / electerm 差异化竞争

**11. Ping/Trace / 网络诊断（~2天）**
- 在监控面板增加延迟/Ping/Traceroute

**12. 拖拽上传（~3天）**
- Finder/资源管理器→终端列自动 SCP

**13. 远程文件编辑（~3天）**
- 双击 SFTP 中的文件→打开内置编辑器/调用系统编辑器

## B.4 不建议做的功能

| 功能 | 原因 |
|------|------|
| RDP/VNC | 不同协议栈，SSH 终端强行做会搞成四不像 |
| Telnet/Serial | 老旧协议，使用率极低 |
| 插件系统 | 投入极大（API+文档+社区），MistTerm 非开源 |
| 云同步/协作 | 需要后端服务，有 Git 同步替代方案 |
| iOS/Android | 移动端开发成本高，不匹配桌面工具定位 |
| 二次认证/生物认证 | 属于服务端策略 |
