1. MistTerm 是什么

MistTerm 是一款 SSH 终端客户端，帮助您：

  • 保存多台服务器连接，一键打开终端
  • 在图形界面里上传、下载文件（SFTP）
  • 保存常用命令模板（片段），少打重复命令
  • 查看服务器 CPU、内存、磁盘等状态
  • 可选：AI 辅助解释命令、团队共享片段

支持 Windows 与 macOS。

2. 安装与启动

2.1 下载安装

  1. 打开 GitHub Releases
  2. 下载对应系统的安装包：
  • Windows：MistTerm-*-windows-x86_64-setup.exe，双击安装
  • macOS：.dmg，拖入「应用程序」文件夹
  1. 从开始菜单 / Launchpad 启动 Mist（或 MistTerm）

安装后无需单独安装 OpenSSH；连接远程 Linux 服务器即可使用。

2.2 首次启动

启动后会出现主窗口：左侧为会话列表，中间为工作区（连接后显示终端），顶部为菜单，底部为状态栏。

图 2-1：已连接服务器时的主界面

2.3 数据保存在哪

连接、密码、片段等配置保存在本机用户目录（已加密），例如：

  • Windows：%APPDATA%\mistterm\
  • macOS：~/Library/Application Support/mistterm/

请勿手动编辑这些文件；在应用内修改即可。

3. 五分钟上手

步骤操作
1按 Ctrl+N（Mac：⌘N）打开「新建会话」
2填写名称、主机 IP/域名、端口（默认 22）、用户名、密码或密钥
3点击 保存并连接，或在侧栏双击该会话
4在中间终端输入命令，如 ls、cd、top
5需要传文件时：菜单 视图 → SFTP，或点击底栏 Files

5. 连接服务器

5.1 新建会话

  1. Ctrl+N 打开对话框
  2. 填写：
  • 会话名称：便于识别，如「生产 Web-01」
  • 主机：IP 或域名
  • 端口：一般为 22
  • 用户名
  • 密码 或 私钥文件路径
  1. 可选：取消勾选「使用 SSH Agent」若只用密码登录
  2. 点击 保存并连接

*图 5-1：也可通过 终端 → 导入 SSH Config 从已有 ~/.ssh/config 批量导入*

5.2 认证方式

方式适用场景
密码服务器允许密码登录
私钥文件本机 .pem / id_ed25519 等
SSH Agent已启动 ssh-agent / Pageant，密钥已加载
跳板机在高级选项中填写 ProxyJump（与 OpenSSH 相同）

5.3 连接与重连

  • 连接：侧栏单击选中 → Ctrl+T 新标签，或双击会话
  • 搜索会话：Ctrl+J 聚焦侧栏搜索框
  • 自动重连：在会话编辑里可开启 KeepAlive / 自动重连
  • 底栏显示 用户名@主机 · 已连接 表示成功

5.4 本地数据安全

密码和会话信息在本地加密保存，换电脑需重新配置或恢复备份。配置文件为加密格式，用记事本打开会看到乱码，属正常现象。

6. 使用终端

6.1 基本操作

  • 连接成功后直接在终端区输入命令，Enter 执行
  • 鼠标选中文字即可复制；Ctrl+V 粘贴（Mac：⌘V）
  • 支持彩色输出、vim、htop 等常见 TUI 程序
  • 标签页：Ctrl+T 新开标签；Ctrl+W 关闭当前标签；Ctrl+Tab 切换标签

7. 传输文件

7.1 SFTP 面板（推荐）

打开方式：菜单 视图 → SFTP，或底栏 Files。

界面分为本机与远程两栏：

  1. 上方输入框可修改当前目录路径，按 Enter 跳转
  2. 单击选中文件或文件夹
  3. 使用面板上的按钮或右键菜单：上传、下载、删除、新建文件夹、重命名
  4. 操作结果在窗口底栏状态显示

图 7-1：SFTP 双栏文件管理

图 7-2：上传完成后底栏提示

典型流程 — 上传文件到服务器：

  1. 连接 SSH 后打开 SFTP 面板
  2. 左侧本机栏进入要上传的文件所在目录
  3. 右侧远程栏进入目标目录
  4. 选中本机文件 → 上传

下载则反向操作：选中远程文件 → 下载到本机当前目录。

7.2 终端内 ZMODEM（rz / sz）

若服务器已安装 lrzsz，可在终端里：

# 从本地上传文件到当前远程目录（执行后按提示选择文件）
rz -bye

# 从远程下载文件到本机
sz 文件名

MistTerm 会自动接管传输并显示进度。若传输异常，请确认远端使用 rz -bye，并检查防火墙是否中断长连接。

9. 监控与端口转发

9.1 主机监控

  1. 先建立 SSH 连接
  2. 打开 视图 → 监控 或底栏 Monitor
  3. 查看 CPU、内存、磁盘、负载等图表（数据来自远程服务器，只读采集）

图 9-1：主机监控

监控依赖 SSH 连接。若连接断开，图表会停止更新；重新连接即可。

9.2 端口转发

  1. 打开底栏 Port Forward 或 视图 → 端口转发
  2. 添加规则（格式与 OpenSSH 类似）：
  • 本地转发：本地端口:目标主机:目标端口
  • 远程转发：远程端口:目标主机:目标端口
  • SOCKS 代理：端口号
  1. 连接会话后规则生效

11. 团队与同步（可选）

面向使用 MistLab 团队版 的用户：

  1. 工具 → 团队账号 登录（浏览器 OAuth 授权）
  2. 登录后可：
  • 同步/共享命令片段

12. 命令安全提示

MistTerm 可在危险命令发送到服务器之前进行提示或拦截（取决于您的规则与团队策略）：

行为说明
拦截如 rm -rf / 等极高风险命令无法发送
确认弹窗询问「是否继续」
告警允许执行但记录告警

可在 偏好设置 中查看或调整本地规则。加入团队后，管理员下发的策略会同步到本机。

13. 凭证、日志与偏好

13.1 凭证管理

工具 → 凭证：集中保存服务器账号、数据库、API Token 等，供会话或片段引用。默认本地加密存储。

13.2 会话日志

在偏好设置中可开启会话日志，自动保存终端交互记录，便于事后审计。可通过 工具 → 浏览会话日志 查看。

13.3 偏好设置与主题

  • Ctrl+, 打开偏好设置：界面语言、主题、字体、审计、日志等

图 13-1：偏好设置

  • Ctrl+H 查看关于与内置快捷键说明

图 13-2：关于 Mist

  • 视图 → 主题：切换深色/浅色等配色

13.4 批量执行

工具 → 批量执行：选择多台已保存的主机，输入同一条命令并行执行，适合巡检（结果分主机显示，过长输出会自动截断）。

15. 常见问题

无法连接服务器

  • 检查主机、端口、用户名、密码/密钥是否正确
  • 确认本机网络能访问目标 IP（可先用系统自带 ssh 命令测试）
  • 若需跳板机，在会话高级选项中配置 ProxyJump
  • 查看底栏是否显示错误信息

连接后终端空白或乱码

  • 稍等几秒等待 shell 初始化
  • 在偏好设置中确认语言与字体；中文环境一般会自动选用合适字体

SFTP 无法上传/下载

  • 确认 SSH 已连接成功（同一标签页）
  • 检查远程目录是否有写权限
  • 本机目标路径是否存在、是否只读

ZMODEM（rz/sz）传文件失败

  • 确认服务器已安装 lrzsz（which rz）
  • 上传时使用 rz -bye
  • 传输过程中勿切换标签或断开连接

配置文件打开是乱码

  • 正常现象：MistTerm 对本地配置做了加密，请只在应用内修改

如何卸载

  • Windows：设置 → 应用 → MistTerm → 卸载
  • macOS：将 MistTerm 拖入废纸篓；配置仍在 Application Support/mistterm/，可手动删除

如何反馈问题

前往 GitHub Issues 描述现象、版本号与操作系统，勿在 issue 中粘贴密码或密钥。
