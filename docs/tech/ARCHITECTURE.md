## 1. 系统概述

### 1.1 产品定位

MistTerm 是一款基于 Rust 和 egui 构建的跨平台 SSH 终端工具，提供：
- 安全的 SSH 连接管理
- 实时终端渲染
- 会话配置持久化
- 多会话并发支持

### 1.2 核心特性

| 特性 | 描述 | 状态 |
|-----|------|-----|
| SSH 连接 | 支持密码认证、密钥认证 | ✅ 已实现 |
| 终端渲染 | ANSI 转义码解析、实时渲染 | ✅ 已实现 |
| 会话管理 | 保存/加载/删除会话配置 | ✅ 已实现 |
| 多会话 | 支持多个并发 SSH 连接 | ✅ 已实现 |
| 跨平台 | macOS、Linux、Windows | ✅ 已实现 |

### 1.3 系统边界


### 2.2 分层架构说明

#### 2.2.1 UI 层 (ui/)

**职责**: 用户界面渲染和交互处理

**核心模块**:
- `app.rs` - 主应用状态管理和生命周期
- 渲染逻辑 - egui 组件树构建
- 事件处理 - 键盘输入、鼠标事件、焦点管理

**依赖**: 核心层

#### 2.2.2 核心层 (core/)

**职责**: 业务逻辑和状态管理

**核心模块**:
- `session.rs` - 会话配置管理（保存/加载/删除）
- `team/`、`fragment_*` 等 - 团队片段、审计、分析
- 消息路由 - SSH 消息分发和处理

**依赖**: SSH 层、终端层

#### 2.2.3 SSH 层 (ssh/)

**职责**: SSH 协议实现和连接管理

**核心模块**:
- `client.rs` - SSH 客户端实现（连接/认证/通道）
- `manager.rs` - 连接池管理（多会话并发）
- 通道管理 - Shell 通道、SFTP 通道

**依赖**: libssh2 C 库

#### 2.2.4 终端层 (terminal/)

**职责**: 终端模拟和输出渲染

**核心模块**:
- `emulator.rs` - ANSI 转义码解析器
- 输出缓冲 - 终端输出缓存和管理
- 渲染适配 - egui 文本渲染

**依赖**: egui

### 3.3 技术选型理由

#### 3.3.1 为什么选择 Rust?

| 优势 | 说明 |
|-----|------|
| **内存安全** | 编译时保证，无数据竞争 |
| **高性能** | 与 C/C++ 相当的性能 |
| **并发安全** | 所有权系统防止数据竞争 |
| **跨平台** | 一次编写，多平台编译 |
| **生态系统** | Cargo 包管理器，丰富的库 |

#### 3.3.2 为什么选择 egui?

| 优势 | 说明 |
|-----|------|
| **即时模式** | 简单高效，易于集成 |
| **跨平台** | 支持所有主流平台 |
| **轻量级** | 无外部依赖，编译快速 |
| **自定义** | 高度可定制的样式和主题 |
| **终端友好** | 适合终端类应用 |

#### 3.3.3 为什么选择 libssh2?

| 优势 | 说明 |
|-----|------|
| **成熟稳定** | 多年维护，广泛使用 |
| **功能完整** | SSH-2 协议完整实现 |
| **广泛支持** | 支持所有主流平台 |
| **性能优秀** | C 库，性能经过优化 |

## 4. 模块设计

### 4.1 UI 层 (ui/app.rs)

#### 4.1.1 核心结构

```rust
pub struct MistTermApp {
    /// 会话管理器
    session_manager: SessionManager,
    /// 连接管理器
    connection_manager: Option<ConnectionManager>,
    /// 消息接收器
    message_rx: Option<Receiver<SshMessage>>,
    /// 选中的会话索引
    selected_session: Option<usize>,
    /// 是否显示连接对话框
    showing_connect_dialog: bool,
    /// 新连接配置
    new_config: SessionConfig,
    /// 输入缓冲
    input_buffer: HashMap<String, String>,
}
```

#### 4.1.2 核心方法

| 方法 | 职责 |
|-----|------|
| `update()` | 应用主循环，处理事件和渲染 |
| `render_header()` | 渲染顶部栏（标题、Connect 按钮） |
| `render_session_list()` | 渲染会话列表 |
| `render_terminal()` | 渲染终端主窗口 |
| `render_input()` | 渲染命令输入框 |
| `render_connect_dialog()` | 渲染连接对话框 |
| `connect_session()` | 发起 SSH 连接 |

#### 4.1.3 事件处理流程

```
用户输入
    │
    ▼
┌─────────────────┐
│  egui 事件循环   │
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  键盘/鼠标事件   │
└─────────────────┘
    │
    ├─► Enter 键 ──┐
    │              ▼
    │        ┌─────────────┐
    │        │ 发送命令    │
    │        │ 到 SSH 通道  │
    │        └─────────────┘
    │
    └─► 点击 Connect ──► 打开连接对话框
```

### 4.2 核心层 (core/)

#### 4.2.1 会话管理 (session.rs)

```rust

    pub name: String,
    pub host: String,
    pub port: u16,
    pub username: String,
    pub password: String,
}

pub struct SessionManager {
    sessions: Vec<SessionConfig>,

```

**核心功能**:
- 加载已保存的会话配置
- 添加新会话
- 删除会话
- 持久化到 JSON 文件

#### 4.2.2 SSH 连接与会话 (ssh/)

连接状态与多会话并发由 **SSH 层**（`ssh/client.rs`、`ssh/manager.rs`）与 UI 中的 `SshSessionHandle` 管理，不再使用已移除的 `core/connection.rs`。

### 4.3 SSH 层 (ssh/)

#### 4.3.1 SSH 客户端 (client.rs)

```rust
pub struct SshConfig {
    pub host: String,
    pub port: u16,
    pub username: String,
    pub password: String,
}

pub struct SshClient {
    config: SshConfig,
    session: Option<Session>,
}
```

**核心方法**:

| 方法 | 职责 |
|-----|------|
| `new()` | 创建客户端实例 |
| `connect()` | 建立 TCP 和 SSH 连接 |
| `authenticate()` | 密码认证 |
| `open_shell()` | 打开 Shell 通道 |
| `send()` | 发送数据到通道 |
| `disconnect()` | 断开连接 |

#### 4.3.2 连接管理器 (manager.rs)

```rust

    next_session_id: usize,
}
```

**核心功能**:
- 管理多个 SSH 客户端实例
- 异步连接处理
- 消息通道管理

## 6. 性能优化

### 6.1 并发设计

```
主线程 (UI)
    │
    ├─► SSH 连接线程 1
    ├─► SSH 连接线程 2
    └─► SSH 连接线程 N

每个 SSH 连接独立线程:
    │
    ├─► 连接建立
    ├─► 认证
    └─► 通道通信
```

**优势**:
- UI 线程不被阻塞
- 多会话并发执行
- 连接失败不影响其他会话

### 6.2 消息传递

使用 `std::sync::mpsc` 进行线程间通信：

```
SSH 线程 ──► Sender ──► Receiver ──► UI 线程
```

**优势**:
- 无锁设计
- 单向数据流
- 易于调试

### 6.3 内存管理

```rust
// 使用 Arc<Mutex<T>> 共享状态
pub struct ConnectionManager {
    sessions: Vec<Arc<Mutex<SshSessionState>>>,
}
```

**优势**:
- 线程安全共享
- 避免数据拷贝
- 细粒度锁

## 7. 安全设计

### 7.1 认证安全

| 认证方式 | 状态 | 说明 |
|---------|-----|------|
| 密码认证 | ✅ 已实现 | 依赖 SSH 传输层加密 |
| 密钥认证 | ✅ 已实现 | 指定私钥路径 + `~/.ssh` 默认密钥 |
| SSH Agent | ✅ 已实现 | ssh-agent / Pageant（会话可开关） |
| 证书认证 | — | **Vault SSH CA 由服务端签发**；桌面端从 Vault KV 读凭证，不实现 CA 签名 |

### 7.2 数据存储

本地敏感 JSON（`sessions.json`、`settings.json`、`fragments.json`、`credentials.json` 等）使用 **`device_key` + AES-256-GCM** 信封加密（格式 `mistterm-aes-v1`）。详见 [SECURITY.md](./SECURITY.md)。

首次启动或升级时，仍为明文 JSON 的旧文件会在加载时自动迁移为加密信封。

### 7.3 连接安全

- ✅ SSH-2 协议（加密传输）
- ✅ 服务器主机密钥验证（`known_hosts` 首次信任）
- ✅ Vault KV 凭证（团队 `vault_credential_path` → 连接前读取密码/私钥）
- 短期 SSH 证书：由 **Vault SSH CA / 运维** 在服务端签发，非桌面客户端职责

## 8. 部署指南

### 8.1 环境要求

| 要求 | 版本 | 说明 |
|-----|------|------|
| Rust | 1.70+ | 编译工具链 |
| libssh2 | 1.9+ | SSH 库 |
| pkg-config | - | 依赖查找 |

### 8.2 编译安装

```bash
# 克隆仓库
git clone https://github.com/mistlab-dev/MistTerm.git
cd MistTerm

# 编译
cargo build --release

# 运行
./target/release/Mist
```

### 8.3 跨平台编译

```bash
# macOS
cargo build --release

# Linux
cargo build --release

# Windows (需要交叉编译环境)
cargo build --release --target x86_64-pc-windows-msvc
```

### 8.4 依赖安装

#### macOS
```bash
brew install libssh2 pkg-config
```

#### Ubuntu/Debian
```bash
sudo apt-get install libssh2-1-dev pkg-config
```

#### CentOS/RHEL
```bash
sudo yum install libssh2-devel pkgconfig
```

#### Windows
使用 vcpkg 或 conan 安装 libssh2
