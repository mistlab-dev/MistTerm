# MistTerm AI 交互设计

> **版本**: 0.2  
> **更新**: 2026-05-20  
> **关联**: [团队平台需求与设计](./TEAM-PLATFORM-DEV-PLAN.md) §5.5（API 与边界）

---

## 1. 一句话

**右侧固定 AI 面板**：生成命令、解释/分析输出都在面板里完成；终端只负责提供可选上下文，以及 **一键把 AI 给出的命令送到左侧终端执行**。

---

## 2. 原则

| 项 | 说明 |
|----|------|
| 模型 | 用户自配 OpenAI 兼容 `base_url` + API Key + `model`（设置页 + 测试连接） |
| 布局 | **一个**右侧 AI 窗口，不另做 `#` 模式、浮层、多入口结果区 |
| 执行 | AI 面板内识别出的命令，用户 **点击「用到终端」** → 写入左侧当前会话输入并 **发送**（等同按 Enter） |
| 上下文 | 可选：把终端**当前选区**附到本条提问（发送前脱敏展示） |
| 团队 | 与团队服务端无关 |

---

## 3. 布局

```text
┌────────┬──────────────────────────────┬─────────────────────┐
│ 会话    │  终端（SSH 输出 + 输入行）      │  AI 面板（右 Dock）   │
│ 侧栏    │  · 拖选输出（不自动复制）       │  ┌─────────────────┐ │
│        │  · 右键「发送到 AI」可选        │  │ 对话 / 回复区    │ │
│        │                              │  │ （生成与解析）    │ │
│        │                              │  ├─────────────────┤ │
│        │                              │  │ 输入框 + 发送     │ │
│        │                              │  └─────────────────┘ │
└────────┴──────────────────────────────┴─────────────────────┘
```

- AI 面板与片段库、监控等 **并列** 为右 Dock 的一项（如标签「AI」），可折叠/拖拽宽度。  
- 未配置 API Key 时：面板提示通过菜单 **「工具 → AI 设置」** 配置，不发起请求。

---

## 4. AI 面板里做什么

所有交互都在面板内，**两种用法，同一输入框**：

1. **生成**：用户输入自然语言（如「查 nginx 最近一小时 5xx 日志条数」）→ 模型回复说明 + 一条或多条命令。  
2. **解析**：用户提问 + 附带终端选区（如报错、日志、`df` 输出）→ 模型回复解读、摘要或分析。

回复区用 **Markdown** 展示；若回复中含 shell 命令（代码块或解析出的命令行），每条命令旁显示：

```text
  journalctl -u nginx --since "1 hour ago" | grep …   [ 用到终端 ]
```

## 5. 用户操作（最小路径）

```text
打开右侧「AI」
  →（可选）终端选中文字 → 右键「发送到 AI」或面板点「附带选区」
  → 在面板底部输入问题 → 发送
  → 阅读回复
  → 点击某条命令旁的 [ 用到终端 ]
  → 左侧当前终端收到该命令并执行
```

### 5.1 「用到终端」

| 行为 | 说明 |
|------|------|
| 写入对象 | 当前焦点 **终端标签页** 的输入行 |
| 发送 | 写入后立即向 PTY **发送**（含回车），等同用户手动 Enter |
| 焦点 | 执行后焦点可留在终端或回到 AI 面板（实现默认：**回终端**） |

不设「仅填入不执行」的二级模式，避免选项过多；若需只复制，用户可复制代码块文本。

### 5.2 附带终端上下文（可选）

| 方式 | 说明 |
|------|------|
| 终端右键 | 「发送到 AI」：把当前选区填入面板「上下文」区（脱敏后可见），再让用户输入问题 |
| 面板按钮 | 「附带当前选区」：无选区时禁用 |

发送前在面板内展示将提交的 excerpt 摘要（可折叠），**不另开预览弹窗**。

### 5.3 脱敏

发送前对上下文做简单替换（Bearer、私钥块、`PASSWORD=` 等 → `[REDACTED]`），在面板上下文区可见。

---

## 6. 终端侧保留什么

| 保留 | 去掉 |
|------|------|
| 拖选高亮 | 输入行 `#` AI 模式 |
| 右键：复制 / 粘贴 / …（现有） | 右键多条「解释/总结/分析」 |
| 右键一项：**发送到 AI**（有选区时） | 命令栏独立 ✨ 浮层 |
| | 独立预览确认弹窗 |
| | 片段库/监控单独 AI 流程（需要时用户在 AI 面板里描述即可） |

---

## 7. 设置（不变）

`enabled`、`base_url`、`api_key`（本机 `settings.json` 内 AES-GCM 加密，不用系统钥匙串）、`model`、`timeout_secs`；**测试连接**；说明文案：请求直连用户填写的服务商。

调用：`POST {base_url}/chat/completions`，面板内多轮对话用 `messages[]` 累积（仅本机，可选清空会话）。

## 8. 实现清单

| # | 项 |
|---|-----|
| 1 | `AiSettings` + 设置页 |
| 2 | 右 Dock「AI」面板（输入 + 对话区 + 上下文区） |

| 6 | 终端右键「发送到 AI」+ 脱敏 |

---

## 9. 验收

1. 配置 API 并测试连接成功。  
2. 在 AI 面板提问生成一条命令，点击「用到终端」，左侧会话出现对应执行。  
3. 终端选中报错 → 发送到 AI → 面板内得到解读。  
4. 未配置 Key 时仅提示去设置。  
5. 未登录团队时 AI 仍可用。
